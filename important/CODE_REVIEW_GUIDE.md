# 🛡️ HappyTool 통합 코드 리뷰 가이드 (Internal Rule)

이 파일은 형님(USER)의 명령에 따라 수행되는 코드 리뷰의 절대 기준입니다. 
`/code-review` 명령이 입력되면 Antigravity는 이 가이드에 따라 변경 사항을 분석합니다.

## 1. 리뷰 범위
- `git status` 및 `git diff`를 통해 파악된 추가/삭제/변경 파일 전체.

## 2. 5대 핵심 리뷰 지표 (형님 수칙)
1. **🚀 App Perf (전역 성능)**: 앱 부팅, 전역 레이아웃, 대규모 상태 변경 등 앱 전체 시스템에 성능 저하를 주는지 확인.
2. **⚡ Change Perf (로직 성능)**: 수정된 로직 내 무거운 연산, 불필요한 리렌더링, API 호출 남발 등을 확인.
3. **🛑 Regression (연쇄 영향)**: 기존 기능이 망가지지는 않았는지, 공통 유틸리티 수정 시 다른 플러그인에 부작용이 없는지 확인.
4. **🔋 RAM & CPU (자원 효율)**: 메모리 누수, 프로세스 점유, 클로저 이슈, 클린업 루틴(Cleanup) 누락 여부 확인.
5. **📏 Big File (파일 크기)**: 파일 하나가 500줄을 넘어가는지 감시. 넘어갈 경우 리팩토링 및 파일 분리 제안.

## 3. 리눅스 개발자 추가 점검 사항 (Antigravity 정신)
- **성능 우선주의**: 워커(Worker) 사용 권장, 메모이제이션 적정성 확인.
- **안정성**: 리그레션 이슈 방지를 위한 방어 코드(Optional Chaining 등) 포함 여부.
- **기록**: `APP_MAP.md` 업데이트 누락 여부.
- **클린 코드**: 매직 넘버 지양, 리눅스 스타일의 간결하고 명확한 코드 지향.

## 4. 리뷰 프로세스
1. 파일 목록 및 등급 산정: [✅ PASS / 👀 CAUTION / ⚠️ WARNING / 🚨 BLOCK]
2. 각 항목별 상세 리포트 제공.
3. 개선 코드 예시(Suggestion) 포함.
