# 원본 보기 가상화 및 줄 수 제한 해제 계획서

형님, 원본 보기에서 로그가 중간에 잘리던 문제를 근본적으로 해결하기 위해 가상화(Virtualization) 기술을 적용하겠습니다.

## 원인 분석
- `PerfTool.worker.ts`에 2,000줄 출력 제한이 걸려 있어, 선택 영역이 길 경우 뒷부분이 생략됨.
- 제한을 푸는 것만으로는 대량의 로그를 한꺼번에 렌더링할 때 브라우저 성능 저하(UI Freeze)가 우려됨.

## 수정 계획
1. **백엔드(Worker) 제한 해제**: `workers/PerfTool.worker.ts`에서 2,000줄로 제한하던 로직을 제거하여 모든 범위를 추출하도록 수정합니다.
2. **프론트엔드 가상화 적용**: `components/PerfTool/index.tsx`의 `PerfRawViewer`에 `react-virtuoso`를 도입합니다.
   - 수만 줄의 로그도 보이는 부분만 렌더링하여 최고의 성능을 유지합니다.
   - `initialTopMostItemIndex`를 사용하여 열자마자 타겟 라인으로 즉시 스크롤되도록 개선합니다.
3. **UI 최적화**: 가상화 환경에 맞춰 줄 번호 영역과 강조(Highlight) 스타일을 더 깔끔하게 다듬습니다.

## 기대 효과
- 아무리 긴 로그 범위라도 끊김 없이 끝까지 확인할 수 있습니다.
- 대량의 로그를 불러와도 메모리 사용량이 최적화되어 브라우저가 버벅이지 않습니다.
