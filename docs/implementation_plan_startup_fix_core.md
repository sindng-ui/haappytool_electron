# 🐧 Startup Latency Optimization Plan (Core Only)

형님, 이전 작업에서 발생했던 이슈들을 고려하여, **UI 로직이나 감성적인 대기 시간(2초)은 건드리지 않고** 오직 기동 시의 **신호 엇박자(20초 대기)**만 해결하는 핵심 수술 계획입니다! 🐧🛡️⚡

## 🎯 목표
1. **백엔드 기동 시간 단축**: 무거운 모듈 지연 로딩 (5초 -> 0.2초).
2. **신호 유실 방지**: 마운트 시점에 백엔드 상태를 직접 조회하여 동기화.
3. **스플래시 탈출 정규화**: 백엔드 준비 완료 시, 2초 대기가 끝나면 즉시 진입하도록 보장.

## 🛠️ 작업 내용

### 1. Backend: Lazy Loading (server/index.cjs)
- `ssh2`, `serialport`, `everythingService` 등 기동 시 무조건 로드되던 무거운 모듈들을 실제 사용 시점에 `require` 하도록 변경.
- **영향 범위**: 기동 속도만 빨라질 뿐, 실제 기능 동작은 동일합니다.

### 2. Frontend: Ready State Sync (App.tsx)
- `useEffect` 마운트 시점에 `window.electronAPI.getStartupStatus()`를 호출.
- 백엔드가 이미 로드 완료 상태라면 `setIsBackendReady(true)`를 즉시 설정하여 신호를 기다리느라 20초를 낭비하는 현상 차단.
- 최소 2초 대기 로직은 `LoadingSplash` 내부에서 그대로 유지되므로 감성 파괴 없음!

### 3. Frontend: Safety Timer Adjustment (App.tsx)
- 백엔드가 준비되었음에도 플러그인 로딩 상태가 업데이트되지 않아 스플래시가 안 닫히는 현상을 방지하기 위해, 백엔드 준비 후 **1초** 뒤에 강제로 진입하도록 설정 (기존 20초 대기 방지).

## 📅 일정
- 계획 승인 후 즉시 수술 시작! (약 5분 소요)

형님, 이 계획대로라면 형님이 원하시는 **"2초간의 가상 로그 감성"**은 지키면서, **"20초간의 답답한 기다림"**만 딱 제거할 수 있슴다! 🐧🚀
