# 로그 분석 에이전트: 사용자 정밀 힌트(PID, TID) 기능 장착! 🔍🎯

형님! 에이전트가 넓은 로그 바다에서 길을 잃지 않도록, 정확한 타겟(PID, TID)과 형님의 주관적인 통찰을 바로 전달할 수 있는 기능을 추가했습니다! 🐧🛡️🛠️✨

## 🛠️ 주요 변경 사항

### 1. 정밀 타격 힌트 필드 추가
- **Process ID (PID) & Thread ID (TID)**: 분석 대상이 되는 프로세스와 스레드 번호를 입력할 수 있는 전용 칸을 마련했습니다. 
- **Additional User Hint**: 형님이 알고 계시는 장애 전후 상황이나 의심되는 점을 자유롭게 적을 수 있는 주관식 입력창(Textarea)을 배치했습니다.
- **UI 위치**: 로그 파일 리스트 바로 아래에 위치하여, 파일을 올린 뒤 자연스럽게 힌트를 적고 분석을 시작할 수 있습니다.

### 2. 에이전트 인지 능력 향상
- **최우선 순위 반영**: 형님이 입력한 정보들은 에이전트에게 전달되는 `initial_hints`의 **가장 첫머리**에 `### [USER HINTS]` 태그와 함께 배치됩니다.
- **분석 집중도 강화**: 에이전트는 시작하자마자 "아, 형님이 1234 프로세스의 5678 스레드를 보라고 하셨구나!"라고 즉시 인지하여 훨씬 빠르고 정확하게 결론에 도달합니다.

---

## 🌟 사용 시 팁
- **PID/TID만 넣어도 OK**: 특정 스레드의 동작만 궁금할 때 TID만 살짝 적어주세요.
- **상황 설명**: "앱 실행 5분 후 네트워크 전환 시 발생" 같은 주관식 힌트는 에이전트에게 엄청난 단서가 됩니다. 🐧🍱
- **비워둬도 OK**: 특별한 정보가 없을 때는 평소처럼 그냥 분석 시작을 누르시면 됩니다!
