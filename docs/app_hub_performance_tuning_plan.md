# 구현 계획서: 앱 허브 애니메이션 및 렌더링 성능 최적화 🐧⚡

디자인의 변경 없이 내부 로직과 CSS 렌더링 방식을 개선하여 앱 허브를 더 가볍고 부드럽게 만듭니다.

## 1. 개선 포인트

### 1.1 Section.tsx: 그리드 레이아웃 연산 최적화
- **`layout="position"` 적용**: `motion.div` (그리드)에 전체 `layout` 대신 `position`만 적용하여 불필요한 크기 재계산(Reflow)을 방지합니다.
- **오케스트레이션 활용**: 개별 카드가 각자 `delay`를 계산하는 대신, 부모 섹션에서 `staggerChildren`을 통해 애니메이션 실행 시점을 효율적으로 배분합니다.

### 1.2 AppCard.tsx: GPU 가속 및 CSS 최적화
- **GPU 레이어 강제**: `transform-gpu` 클래스를 추가하여 브라우저가 카드를 별도의 컴포지팅 레이어로 분리하게 합니다.
- **`will-change` 확장**: `filter` 속성을 추가하여 `backdrop-blur` 연산 시 GPU가 미리 준비할 수 있도록 합니다.
- **CSS Transition 정밀화**: `transition-all`을 제거하고 필요한 속성(`background-color`, `border-color`, `box-shadow`)만 감시하도록 수정합니다.

### 1.3 렌더링 전략
- **안정적 초기화**: 모달이 열리는 찰나의 순간에 모든 카드가 `layout` 측정을 시작하지 않도록, 등장 애니메이션과 레이아웃 추적 로직을 조율합니다.

## 2. 기대 효과
- **프레임 드랍(Jank) 감소**: 수십 개의 카드가 동시에 뜰 때 발생하는 순간적인 멈춤 현상을 해결합니다.
- **GPU 점유율 최적화**: 유리 질감 효과가 적용된 상태에서도 애니메이션이 60fps에 가깝게 동작하도록 개선합니다.
