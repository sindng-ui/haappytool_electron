# NetTraffic 검색 기능 추가 및 추출 로직 정밀화 계획

형님! 특정 주소가 안 보인다면, 제가 검색창을 만들어서 바로 찾아내실 수 있게 해드리겠습니다! 🐧⚡
또한, 정규식이 URL 앞뒤의 군더더기(예: `URI:`, `URL:`)를 같이 잡아버려서 목록에서 찾기 힘드셨을 수도 있으니 추출 로직도 더 정교하게 다듬겠습니다.

## 주요 개선 사항

### 1. UI 검색 기능 추가 (`NetTrafficAnalyzerView.tsx`) 🔍
- **Endpoints 탭**: 상단에 검색 바를 추가하여 URI나 에일리어스로 즉시 필터링할 수 있게 합니다.
- **UA 탭**: 클라이언트 정보나 엔드포인트명으로 검색할 수 있는 기능을 추가합니다.

### 2. URI 추출 로직 정밀화 (`NetTraffic.worker.ts`) 🎯
- **경계 조건 강화**: URL이 `URI:http...` 처럼 붙어 있는 경우에도 순수하게 `http...` 부분만 잘 추출하도록 정규식을 개선합니다.
- **대소문자 무관 검색 강화**: 이미 적용했지만, 한 번 더 꼼꼼하게 살피겠습니다.

### 3. 기본 규칙 키워드 비우기 🕊️
- 새로운 규칙을 만들 때 기본 키워드를 아예 비워두어, 추가 설정 없이도 모든 로그에서 엔드포인트를 뽑아낼 수 있도록 유도합니다. (현재 `ST_APP`이 들어있어 이를 모르는 경우 필터링이 될 수 있음)

## 검증 계획

### 수동 검증
- 형님이 주신 `https://up.stdive.com/sup/fmx/device/list` 샘플 로그를 검색 바에 입력하여 즉시 나타나는지 확인합니다.
- 다양한 접두사(`URL:`, `(https://...)`)가 붙은 로그에서도 주소만 잘 뽑히는지 확인합니다.
