# Happy Combo 사용성 개선 구현 계획서 🚀

형님! Happy Combo가 많아졌을 때의 피로도를 획기적으로 줄여줄 개선 계획입니다. 🐧⚡

## 1. 개요
현재 Log Extractor의 Happy Combo 설정 섹션에서 콤보 개수가 많아지면 신규 콤보 추가나 관리가 어려워지는 문제를 해결합니다.

## 2. 주요 개선 사항

### 2.1. Sticky Header & Quick Add Button 📌
- **문제**: 신규 콤보 추가 버튼이 리스트 최하단에 있어 스크롤이 필요함.
- **해결**: Happy Combo 섹션의 헤더를 상단에 고정(Sticky)하고, 헤더 내에 즉시 추가할 수 있는 `+` 버튼을 배치합니다.

### 2.2. Combo Search & Filter 🔍
- **문제**: 콤보가 수십 개일 때 특정 콤보를 찾기 어려움.
- **해결**: 헤더 영역에 검색창을 추가하여 콤보 이름(Root Name)이나 포함된 태그로 필터링할 수 있게 합니다.

### 2.3. Collapse/Expand All 📦
- **문제**: 모든 콤보가 펼쳐져 있으면 화면이 너무 길어짐.
- **해결**: 전체 콤보를 한 번에 접거나 펼 수 있는 일괄 토글 버튼을 추가합니다.

### 2.4. Local Add Branch Button ➕
- **문제**: 특정 콤보 내에서 브랜치를 추가할 때도 해당 콤보의 최하단으로 가야 함.
- **해결**: 각 콤보 그룹의 Root 이름 바로 옆에 작은 `+` 아이콘을 배치하여 즉시 브랜치를 추가할 수 있게 합니다.

## 3. 상세 구현 계획

### 3.1. `HappyComboSection.tsx` 수정
- [ ] **Sticky Header 구성**: 
  - `Happy Combos` 타이틀, 검색창, 일괄 접기/펴기, 퀵 추가 버튼을 포함하는 Sticky Header 구현.
  - 스크롤 시 배경색과 그림자를 주어 리스트와 구분되게 처리.
- [ ] **필터링 로직 추가**: 
  - `searchTerm` 상태를 추가하고, `groupedRoots`를 렌더링하기 전에 필터링.
- [ ] **일괄 접기/펴기 기능**: 
  - `setCollapsedRoots`를 호출하여 모든 Root를 `collapsedRoots`에 넣거나 빼는 함수 구현.
- [ ] **개별 그룹 내 퀵 브랜치 추가**: 
  - Root Header에 `Plus` 아이콘 버튼 추가.

### 3.2. UX/UI 폴리싱 ✨
- **Glassmorphism**: 기존 디자인 시스템에 맞춰 유리 질감과 은은한 애니메이션 적용.
- **Highlighting**: 검색 결과와 일치하는 키워드가 있을 경우 시각적으로 강조.

## 4. 기대 효과
- **추가 속도 2배 이상 향상**: 스크롤 없이 어디서든 즉시 추가 가능.
- **관리 편의성 극대화**: 검색과 일괄 접기를 통해 수백 개의 콤보도 한눈에 관리.
