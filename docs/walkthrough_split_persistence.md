# Split 모드 상태 유지 및 자동 복원 완료 (Walkthrough)

형님! 이제 앱을 껐다 켜도 형님이 보고 계시던 Split 모드와 양쪽 로그 파일이 그대로 살아납니다! 🐧🚀

## 🌟 주요 작업 내용

### 1. Split 모드 영속화 (Persistence) 💾
- **탭별 상태 저장**: `localStorage`(`tabState_${tabId}`)에 `isDualView` 상태와 `rightFilePath`를 추가로 저장하도록 개선했습니다.
- **주기적 동기화**: 1초마다 저장되는 로직에 Split 모드 정보를 포함시켜 갑작스러운 종료에도 대응합니다.

### 2. 자동 복원 (Auto Restoration) 🔄
- **지능형 로딩**: 탭 로드 시 `isDualView`가 활성화되어 있으면, 좌측 파일뿐만 아니라 우측 파일도 자동으로 서버에서 불러와 워커를 초기화합니다.
- **안정성 확보**: 파일 로딩 시 중복 요청을 방지하고, Pane별로 독립적인 로딩 상태를 관리하도록 `useLogFileOperations`를 리팩토링했습니다.

### 3. 버그 수정 및 안정화 🛠️
- **로그 닫기 반영**: 로그 닫기(X) 버튼을 눌렀을 때 파일 경로 상태를 명시적으로 비위, 재시작 시 닫았던 로그가 살아나지 않도록 수정했습니다.
- **Single 모드 수복**: `LogExtractor`에서 탭 복구 시 파일 경로가 유실되던 문제를 해결하여 Single 모드에서도 자동 로딩이 정상 작동하도록 했습니다.
- **레이스 컨디션 해결**: 부팅 시 데이터가 로드되기 전에 빈 상태로 덮어씌워지는 현상을 방지하기 위해 `isLoaded` 플래그를 도입했습니다.
- **스트림 분리**: 좌/우 로그 로딩이 겹칠 때 데이터가 섞이지 않도록 Request ID를 독립적으로 분리했습니다.
