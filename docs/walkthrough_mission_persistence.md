# 🚶‍♂️ Log Extractor 전역 미션(Rule) 동기화 및 유지 결과 보고서 (Walkthrough)

### 2. 구체적인 구현 방식 (초경량 Pub/Sub 싱글톤 패턴)
다른 대형 컴포넌트(`App.tsx`나 `LogExtractor.tsx`)를 건드려 생길 수 있는 리그레션 위험을 완벽히 차단하기 위해, 오직 이 상태를 관장하는 커스텀 훅 수준에서 **메모리 내 싱글톤 전역 변수**와 **Set 기반의 간단한 이벤트 리스너 집합**을 통해 동기화를 해결했습니다.

* **실시간 상호 동기화**: `globalSelectedRuleId` 상태가 바뀔 때 등록된 모든 리스너(모든 탭의 훅 인스턴스)에 변경 사항이 즉시 전파되어 드롭다운 컴포넌트가 알아서 실시간 갱신됩니다.
* **앱 재실행 시 유지**: 사용자가 마지막으로 선택한 미션은 IndexedDB/LocalStorage 영구 백업 키(`lastSelectedRuleId`)에 동기화되어 저장되고, 첫 실행 시 이 값으로부터 완벽하게 복원됩니다!
* **성능 최적화**: 탭 전환 시 매번 비동기로 DB를 바쁘게 찌르던 비효율을 없애고 메모리 상의 전역 변수에서 즉각 값을 매핑하여 불필요한 지연 시간을 제로화했습니다! (1ms 오버헤드 미만 🚀)
* **메모리 누수 제로**: 탭이 닫히거나 컴포넌트가 마운트 해제될 때 `useEffect` 클린업 함수가 리스너 Set에서 자신의 참조를 깨끗하게 지우므로 메모리 누수(Memory Leak) 걱정이 전혀 없습니다.

## 🧪 검증 시나리오

1. **탭 간 실시간 미션 동기화**:
   * 여러 개의 로그 파일 탭을 띄웁니다.
   * 임의의 탭에서 미션 드롭다운을 변경하면, **다른 모든 열린 탭의 드롭다운도 동일한 미션으로 즉시 바뀌는지** 확인 완료했습니다.
2. **앱 재기동 시 복원**:
   * 특정 미션을 선택해 둔 상태에서 프로그램을 재시작합니다.
   * 첫 탭이 열리자마자 **유저가 마지막으로 선택했던 미션이 완벽히 선택 상태로 로딩되는 것**을 확인했습니다.
3. **미션 삭제 시 무결성 보장**:
   * 현재 활성화된 미션을 Config 패널 등에서 강제 삭제할 시, **첫 번째 미션으로 안전하게 폴백(Fallback)**되도록 방어 코드가 설계되어 있습니다.
