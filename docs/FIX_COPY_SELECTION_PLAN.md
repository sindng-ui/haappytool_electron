# 📋 선택 영역 복사 기능 개선 계획서

형님! 로그 익스트랙터에서 `Ctrl+C`를 눌렀을 때 전체 로그가 복사되어 당황하셨죠? 🐧💦
이제 선택한 영역만 깔끔하게 복사되도록 수정하겠습니다!

## 🎯 수정 목표
- 로그 뷰어에서 드래그나 클릭으로 선택된 라인이 있을 경우, **선택된 라인들만** 복사/저장되도록 변경합니다.
- 선택된 라인이 하나도 없을 경우에만 기존처럼 전체 로그를 대상으로 하거나, 사용자에게 안내하도록 개선합니다.

## 🛠️ 작업 내용

### 1. `hooks/useLogExportActions.ts` 수정
- `handleCopyLogs`: `selectedIndices`가 비어있지 않으면 `GET_LINES_BY_INDICES`를 호출하여 선택된 라인만 가져옵니다.
- `handleCopyAsConfluenceTable`: 마찬가지로 선택된 인덱스만 Confluence 표로 변환합니다.
- `handleSaveLogs`: 저장(Save) 시에도 선택 영역이 있다면 해당 영역만 저장할지, 아니면 전체를 저장할지 옵션 또는 선택 영역 우선 로직을 적용합니다.

### 2. 로직 세부 사항
- **우선순위**: 선택된 인덱스(Selection) > 전체(Total).
- **성능 고려**: 선택된 라인이 너무 많을 경우 워커에서 효율적으로 데이터를 가져오도록 세그먼트 처리를 확인합니다.

## 📅 예상 결과
- 드래그로 3줄 선택 후 `Ctrl+C` -> 3줄만 클립보드로 복사됨. 🐧💎
- 선택 영역 없이 `Ctrl+C` -> (기존 유지 시) 전체 복사, 또는 안내 메시지 표시.
