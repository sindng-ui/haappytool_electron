# Analyze Diff: Sequence Matching 엔진 구현 완료 🐧🧩🚀

형님! 드디어 **Analyze Diff** 기능의 핵심인 **시퀀스 매칭(Sequence Matching)** 엔진 구현을 완료했습니다. 이제 로그의 순서가 조금 뒤섞이거나 중간에 다른 로그가 끼어들어도, Happy Combo Alias를 기준으로 정확하게 대응 지점을 찾아 시간 차이를 비교할 수 있습니다!

## 🛠️ 주요 변경 사항

### 1. Alias 이벤트 추출 엔진 (Log Worker)
- `extractAliasEvents` 시스템 도입: 필터링된 모든 로그를 스캔하여 Happy Combo Alias가 매칭되는 지점을 타임스탬프와 함께 추출합니다.
- 초거대 로그 대응: 1GB 이상의 로컬 파일이나 실시간 스트림에서도 효율적으로 Alias를 추출할 수 있도록 최적화되었습니다.

### 2. 순서 기반 매칭 알고리즘 (Split Analysis Worker)
- **N번째 vs N번째 매칭**: 왼쪽 로그의 N번째 'AppLaunch'와 오른쪽 로그의 N번째 'AppLaunch'를 매칭하는 지능형 시퀀스 알고리즘을 구현했습니다.
- **차이점 가시화**:
  - `[Alias Match]`: 매칭 성공 시 두 이벤트 사이의 시간 차이(Delta)를 표시합니다.
  - `[Alias New]`: 오른쪽에만 존재하는 새로운 핵심 이벤트를 별도로 분류합니다.

### 3. 데이터 파이프라인 연동 (useSplitAnalysis Hook)
- 메트릭 분석과 Alias 추출을 **병렬(Parallel)**로 실행하여 대용량 파일 분석 속도를 극대화했습니다.
- 추출된 데이터를 분석 워커로 전달하여 최종 리포트를 생성합니다.

## 🗺️ APP_MAP 업데이트
- `[[Analyze Diff & Sequence Matching]]` 섹션을 추가하여 새로운 아키텍처와 데이터 흐름을 명시했습니다.
- 이제 AI Agent가 로그 비교 로직을 훨씬 더 명확하게 이해하고 확장할 수 있습니다.
