# App Hub "버벅임 제로" & "엇박자 감성" 최적화 완료

형님! "감성"은 지키면서 "버벅임"만 싹 걷어내는 고난도 수술을 마쳤습니다. 🐧⚡

## 🛠️ 주요 변경 사항

### 1. React Memoization 완전 복구 (성능 핵심)
- **문제**: 부모(`Section`)에서 자식(`AppCard`)으로 함수를 넘길 때 인라인 화살표 함수를 사용하여, `React.memo`가 무력화되고 모든 카드가 무한 리렌더링되고 있었습니다.
- **해결**: `AppCard` 인터페이스를 수정하여 안정된 함수 참조를 받도록 변경했습니다. 이제 모달이 열릴 때 카드가 딱 한 번씩만 그려지므로 CPU 부하가 극적으로 줄어듭니다.

### 2. 무거운 Framer Motion 기능 제거
- **`layout` 속성 삭제**: 그리드 내에서 수십 개의 카드 위치를 실시간 추적하던 `layout` 기능을 걷어내어 브라우저의 레이아웃 계산 부하를 제거했습니다.
- **`layoutId` 삭제**: 활성화 표시기에 사용되던 복잡한 좌표 동기화 로직을 가벼운 CSS 효과로 대체했습니다.

### 3. "엇박자" & "굼뜬" 감성 복구
- **초기 대기시간 유지**: 모달이 열린 후 첫 카드가 나오기 전까지의 그 미묘한 정적(`delayChildren: 0.25s`)을 그대로 살렸습니다.
- **무작위 리듬감**: 각 카드마다 고유의 ID를 기반으로 한 무작위 지연시간(`randomFactor.delay`)을 적용하여, 카드가 동시에 뜨지 않고 엇박자로 기분 좋게 쏟아지는 느낌을 유지했습니다.
- **Spring 애니메이션**: 탄성 있는 Spring 모델을 사용하여 등장할 때는 찰지게, 멈출 때는 부드럽게 감속되도록 조정했습니다.

- **체감 성능**: 저사양 환경에서도 렉 없이 부드러운 엇박자 애니메이션 구현
