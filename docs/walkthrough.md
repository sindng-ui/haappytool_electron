# 워크스루: CLI 유닛 테스트 안정화 🐧🔧

형님, CLI 유닛 테스트 중에 `block-test` 시나리오가 5초 타임아웃으로 자꾸 실패하던 녀석을 드디어 잡았습니다! 이제 모든 테스트가 아주 쾌속으로 통과됩니다.

## 1. 문제 원인 분석
- `CliBlockTestWrapper` 컴포넌트 내부에서 `setTimeout`을 사용하여 3초간 소켓 연결을 기다리는데, 테스트 코드에서 가상 타이머(`vi.useFakeTimers()`)가 렌더링 이후에 활성화되어 실제 타이머와 충돌이 발생했습니다.
- 이로 인해 `waitFor`가 가상 시간의 흐름을 제대로 인지하지 못하고 실제 시간 5초가 지날 때까지 대기하다가 타임아웃이 발생한 것이었습니다.

## 2. 해결 방법
- **가상 타이머 전면 도입**: `describe` 블록의 `beforeEach`에서 `vi.useFakeTimers()`를 호출하여 컴포넌트가 마운트되기 전부터 타이머를 장악했습니다.
- **테스트 최적화**: 중복된 `renderCliApp()` 호출을 제거하고, `act` 블록 내에서 `vi.advanceTimersByTime(3500)`을 호출하여 3.5초의 시간을 즉시 가속했습니다.
- **안정성 강화**: `waitFor` 대신 명시적인 `expect`와 `async act`를 사용하여 마이크로태스크(Async callback)까지 확실히 처리한 뒤 결과를 확인하도록 변경했습니다.

## 3. 결과 확인
- **속도 향상**: 기존에 최소 5초 이상 걸리던(혹은 실패하던) 테스트가 **92ms** 만에 끝납니다! 약 **50배 이상의 성능 향상**을 이뤄냈습니다. 🚀
- **전체 성공**: 34개의 테스트 파일(271개 테스트)이 모두 성공적으로 통과되었습니다. (1개 파일 스킵 제외)
