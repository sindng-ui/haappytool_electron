# [구현 계획] LogQuickTagsPopover 크기 확장 및 UI 고도화 🚀

형님! 요청하신 대로 TopBar에서 로깅 태그를 팍팍 제어할 수 있는 `LogQuickTagsPopover`의 크기를 인심 좋게 대폭 넓히고, 한층 더 고급스럽고 시원시원한 프리미엄 UI로 개선하는 계획서입니다! 🐧⚡

날짜가 변경되었거나 새로운 대화가 시작되어 `important/` 폴더 내에 있는 모든 최신 아티팩트와 지도(`APP_MAP.md`)를 확실히 완벽하게 숙지하고 작업에 들어갑니다!

## 1. 형님 검토 요청 사항 (User Review Required)

> [!IMPORTANT]
> - **크기 대폭 확장 (`w-72` ➔ `w-[480px]`)**: 기존의 너무 좁고 답답했던 `w-72` (288px) 너비를 **`w-[480px]` (또는 형님 취향에 맞게 조정 가능)**로 대폭 넓혀 많은 로그 태그를 한눈에 여유롭게 보고 편집할 수 있도록 합니다.
> - **레이아웃 비주얼 극대화**: 태그들이 들어가는 칩 입력창의 최소 높이를 `min-h-[44px]`에서 **`min-h-[96px]`**로 시원하게 키우고 패딩을 넉넉하게 주어 태그 입력 시 쾌적한 피드백을 제공합니다.
> - **안전한 무결성 유지**: 로깅 시작/정지 명령 전송, 타이젠(Tizen) 단말 연결 예외 처리 로직 등 백엔드 핵심 코드는 100% 보존하면서 리그레션 이슈가 전혀 생기지 않도록 UI 레이아웃 요소 위주로 수정합니다.
> - **성능 최적화**: 팝오버 렌더링 시 성능을 무겁게 하는 `blur` 필터는 절대 사용하지 않고(규칙 준수), GPU 가속에 용이한 깔끔한 반투명 다크 배경과 강력하고 부드러운 그림자 효과(`shadow-[0_24px_64px_rgba(0,0,0,0.7)]`)를 조합하여 미려한 디자인을 뽑아내겠습니다!

## 2. 세부 변경 사항 (Proposed Changes)

### [[Log Viewer UI Component]]

#### [MODIFY] [LogQuickTagsPopover.tsx](file:///k:/Antigravity_Projects/gitbase/happytool_electron/components/LogViewer/LogQuickTagsPopover.tsx)

- **팝오버 너비 확장**: `w-72` ➔ `w-[480px]` (더 시원하게 배치)
- **헤더 영역 확대**: 패딩을 `px-4 py-3`에서 `px-5 py-4`로 넓히고 글자 크기 가독성 개선, 아이콘 크기도 `size={13}`에서 `size={15}`로 확대하여 더욱 당당하게 배치합니다.
- **태그 칩 컨테이너 확대**: `min-h-[44px]`에서 `min-h-[96px]`로 확장하고, 내부 여백을 주어 타이핑 및 칩이 나열될 때의 시각적 답답함을 근절합니다.
- **입력 칩 & 인풋 고도화**:
  - 태그 칩 패딩: `px-2 py-0.5` ➔ `px-3 py-1.5`로 키우고 둥근 모서리와 은은한 테두리를 적용하여 해피콤보 칩 스타일과 완전히 일치시킵니다.
  - 삭제 아이콘 크기: `size={9}` ➔ `size={11}`로 키워 클릭 영역과 정확도를 향상시킵니다.
  - 인풋 폰트 및 패딩 개선: `text-xs`의 가독성을 높이고 모노스페이스 스타일링을 정돈합니다.
- **Start / Stop Logging 버튼 영역 확대**:
  - 버튼 패딩 및 높이: `py-2` ➔ `py-3 text-sm`로 확장하여 한눈에 들어오는 강력한 액션 버튼으로 강화합니다.
  - `Play`/`Square` 아이콘 크기: `size={13}` ➔ `size={15}`
- **안내 메시지 가독성**: 단말 미연결 시 표시되는 경고 배너도 한눈에 보이도록 가독성을 높입니다.
