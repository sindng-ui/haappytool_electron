## 📊 Log Extractor - 로그 분석 도구

**로그 파일에서 원하는 내용만 빠르게 찾아내는 강력한 분석 도구**

### 주요 기능

#### 1️⃣ 로그 파일 불러오기
- **드래그 앤 드롭**: 로그 파일을 화면에 드래그하면 자동으로 분석 시작
- **파일 선택**: 상단의 파일 아이콘을 클릭하여 파일 선택
- **대용량 파일 지원**: 수백만 줄의 로그도 빠르게 처리

#### 2️⃣ 필터링 규칙 (Happy Combos)
**원하는 로그만 골라내는 강력한 필터**

- **Root (최상위 검색어)**: 반드시 포함되어야 하는 키워드
  - 예: `error`, `warning`, `success`
  
- **Branch (하위 검색어)**: Root와 함께 포함되어야 하는 추가 키워드
  - 예: Root가 "error"일 때, Branch로 "network", "timeout" 추가
  
- **여러 조합 만들기**: `+ New Combo Tree` 버튼으로 다양한 필터 조합 생성
  - OR 조건: 여러 Root를 만들면 OR 조건
  - AND 조건: 하나의 Root에 여러 Branch를 추가하면 AND 조건

#### Happy Combo 편집 팁 (New)
**더 빠르고 편리한 태그 편집**

- **Backspace (지우기)**:
  - 빈 태그에서 `Backspace`: 현재 태그 삭제 후 이전 태그로 이동
  - 빈 브랜치에서 `Backspace`: 브랜치 삭제 후 이전 브랜치로 이동
  - **마지막 브랜치 보호**: 마지막 남은 브랜치를 지우면 완전히 삭제되지 않고 Root만 남음

- **Enter (입력)**:
  - 태그 입력 후 `Enter`: 수정 완료
  - 마지막 태그에서 `Enter`: 자동으로 새 태그 입력 칸(`+ tag`)으로 이동

- **방향키**:
  - `↑` / `↓`: 브랜치 간 빠른 이동

#### 3️⃣ 차단 목록 (Block List)
**보고 싶지 않은 로그 걸러내기**

- 차단하고 싶은 단어 입력
- 해당 단어가 포함된 로그는 결과에서 제외
- 예: "debug", "verbose" 등

#### 4️⃣ 색상 강조 (Color Highlights)
**중요한 키워드를 색깔로 표시**

- 강조하고 싶은 단어 입력
- 7가지 색상 중 선택
- 로그에서 해당 단어가 색상으로 강조 표시

#### 5️⃣ 듀얼 뷰 (Split View)
**두 개의 로그를 나란히 비교**

- 상단의 `듀얼 뷰` 버튼 클릭
- 왼쪽/오른쪽에 각각 다른 로그 파일 열기
- 같은 문제를 다른 환경에서 비교할 때 유용

#### 6️⃣ 원본 로그 보기
**필터링된 로그의 원본 위치 확인**

- 로그 줄을 **더블클릭**
- 하단에 원본 파일의 전체 내용 표시
- 앞뒤 맥락 파악에 유용

#### 7️⃣ 북마크
**중요한 로그 줄 표시**

- 로그 줄 왼쪽의 `⭐` 아이콘 클릭
- 노란색으로 강조 표시
- 나중에 다시 찾기 쉽게

#### 8️⃣ 로그 저장 및 복사
- **Copy Logs**: 필터링된 로그를 클립보드에 복사
- **Save Logs**: 필터링된 로그를 파일로 저장

### 키보드 단축키 (Log Extractor)
- `Ctrl + ↑/↓`: 한 줄씩 스크롤
- `PageUp/PageDown`: 페이지 단위 스크롤 (커서 위치 기반 스마트 스크롤)
- `Ctrl + ←/→`: 듀얼 뷰에서 좌우 패널 전환
- `Ctrl + 0`: 줌 리셋 (100%)
- `Ctrl + +/-`: 줌 인/아웃

## 🚀 POST Tool - API 테스트 도구

**REST API를 간편하게 테스트하는 도구 (Postman 대체)**

### 주요 기능

#### 1️⃣ HTTP 요청 보내기
- **메소드 선택**: GET, POST, PUT, DELETE, PATCH 지원
- **URL 입력**: API 엔드포인트 주소 입력
- **헤더 설정**: 
  - `Content-Type`, `Authorization` 등 자유롭게 추가
  - Key-Value 형식으로 입력
- **바디 입력** (POST/PUT/PATCH):
  - JSON, XML, 텍스트 등 자유 형식
  - 실시간 문법 검사

#### 2️⃣ 응답 확인
- **상태 코드**: 200, 404, 500 등 HTTP 상태
- **응답 시간**: 요청부터 응답까지 걸린 시간 (ms)
- **응답 헤더**: 서버가 보낸 헤더 정보
- **응답 바디**: 
  - JSON 자동 포맷팅
  - 문법 강조 표시

#### 3️⃣ 요청 저장 및 관리
- **저장**: 자주 사용하는 요청을 저장
- **이름 지정**: 각 요청에 알기 쉬운 이름 부여
- **빠른 전환**: 왼쪽 사이드바에서 클릭 한 번으로 요청 전환

### 사용 예시
```
1. 새 요청 만들기 (+ 버튼)
2. 메소드를 POST로 선택
3. URL: https://api.example.com/users
4. 헤더 추가: Content-Type: application/json
5. 바디 입력:
   {
     "name": "홍길동",
     "email": "hong@example.com"
   }
6. Send 버튼 클릭
7. 응답 확인
```

## 🔧 JSON Tools - JSON 처리 도구

**JSON 데이터를 쉽게 다루는 도구**

### 주요 기능

#### 1️⃣ JSON Formatter & Validator (형식 변환 & 검증)
**JSON 데이터 정리 및 오류 확인**

- **붙여넣기**: JSON 텍스트를 왼쪽에 붙여넣기
- **자동 포맷팅**: 
  - 들여쓰기 자동 정렬
  - 읽기 쉬운 형태로 변환
- **오류 검사**:
  - 문법 오류 자동 감지
  - 오류 위치 표시
- **복사**: 정리된 JSON을 클릭 한 번으로 복사

**사용 사례**:
- API 응답을 보기 좋게 정리
- JSON 파일의 문법 오류 찾기
- 압축된 JSON을 읽기 쉽게 펼치기

#### 2️⃣ JSON Diff Viewer (비교 도구)
**두 개의 JSON을 비교하여 차이점 찾기**

- **좌/우 입력**: 비교할 두 JSON을 각각 입력
- **차이점 강조**:
  - 🔴 빨간색: 삭제된 항목
  - 🟢 초록색: 추가된 항목
  - 🟡 노란색: 변경된 항목
- **줄 번호 표시**: 정확한 위치 파악

**사용 사례**:
- 설정 파일 변경 사항 확인
- API 응답 버전 비교
- 백업 파일과 현재 파일 비교

## 📦 TPK Extractor - 패키지 추출 도구

**Tizen RPM 패키지에서 TPK 파일 추출**

### 주요 기능

#### 1️⃣ RPM 파일 변환
- **.rpm 파일 선택**: 드래그 앤 드롭 또는 클릭하여 선택
- **자동 변환**: RPM 내부의 TPK 파일 자동 추출
- **다운로드**: 추출된 TPK 파일 저장

#### 2️⃣ 진행 상황 표시
- 실시간 진행률 표시
- 추출 성공/실패 메시지

### 사용 방법
```
1. RPM 파일을 화면에 드래그 앤 드롭
2. 자동으로 추출 시작
3. 완료되면 Download 버튼 클릭
4. TPK 파일 저장 위치 선택
```

## 💡 팁과 요령

### Log Extractor
1. **복잡한 조건은 여러 Combo로**: 하나의 복잡한 조건보다 여러 개의 간단한 Combo가 관리하기 쉽습니다
2. **색상을 적극 활용**: 중요도에 따라 다른 색상 부여
3. **북마크로 임시 표시**: 나중에 다시 봐야 할 로그는 북마크
4. **듀얼 뷰 활용**: 문제가 발생한 로그와 정상 로그를 나란히 비교

### POST Tool
1. **자주 쓰는 요청은 저장**: 매번 입력하지 말고 저장해서 재사용
2. **헤더 미리 설정**: Authorization 같은 공통 헤더는 미리 설정

### JSON Tools
1. **붙여넣기 전 확인**: 큰 JSON은 에디터에서 일부만 복사하여 테스트
2. **Diff는 포맷팅 후**: 비교하기 전에 양쪽 모두 포맷팅하면 더 정확

## ❓ 자주 묻는 질문 (FAQ)

**Q: 로그 파일이 너무 커서 느려요**
A: HappyTool은 대용량 파일도 처리할 수 있지만, 수십 GB 파일은 시간이 걸립니다. 가능하면 필요한 부분만 추출하여 사용하세요.

**Q: 필터링 결과가 안 나와요**
A: 
- Block List에 너무 광범위한 단어가 있는지 확인
- Root와 Branch 조건이 너무 엄격한지 확인
- 대소문자 구분을 확인하세요

**Q: 설정을 저장하고 싶어요**
A: Settings에서 Export를 클릭하여 JSON 파일로 저장 가능합니다. Import로 다시 불러올 수 있습니다.

**Q: 다크 모드를 끄고 싶어요**
A: 현재 버전은 다크 모드 고정입니다. 눈의 피로를 줄이기 위한 선택입니다.

## 🎨 UI 요소 설명

### Configuration 패널
- **접기/펴기 버튼**: 오른쪽 상단의 네모난 탭 버튼 (화살표 아이콘 포함)
  - 패널이 열려있을 때: < 아이콘 (왼쪽 화살표)
  - 패널이 닫혀있을 때: > 아이콘 (오른쪽 화살표)
- **크기 조절**: 패널 오른쪽 경계선을 드래그하여 크기 조절

### 로그 뷰어
- **줄 번호**: 왼쪽에 표시되는 숫자 (원본 파일의 줄 번호)
- **북마크**: 줄 번호 옆의 별 아이콘
- **스크롤바**: 오른쪽 스크롤로 빠른 이동

## 📞 문제 해결

### 앱이 실행되지 않아요
1. Windows Defender가 차단했는지 확인
2. 관리자 권한으로 실행
3. 재설치

### 파일을 열 수 없어요
1. 파일 경로에 한글이나 특수문자가 있는지 확인
2. 파일이 다른 프로그램에서 열려있는지 확인
3. 읽기 권한이 있는지 확인

### 기능이 작동하지 않아요
1. 앱을 완전히 종료 후 재시작
2. 최신 버전인지 확인
3. Developer Tools (F12)에서 에러 메시지 확인
